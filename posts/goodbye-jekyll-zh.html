再見了，Jekyll

JekyllGitHub PagesStatic SiteAI

其他語言：English

謝謝 AI，我終於擺脫 Jekyll 了。

大概七八年前，我第一次知道 GitHub 提供免費個人網站。興奮了一陣之後，我被複雜的前後端設定打得落花流水。

後來我遇見 Jekyll。是在某個 GitHub 頁面上撞見的。我注意到頁面底部有一行小字，寫著 “Powered by Jekyll。”我就像沙漠裡快渴死的人看見綠洲，立刻撲過去。外面有很多漂亮的版型，第一次架起來快得不得了，但沒多久細節就開始浮現。

雖然說它應該讓你不必管前端程式和 CSS 語法，專心寫內容和調樣式，你還是得花時間找一個自己看得順眼的版型。好不容易選好了，又會發現還有不少地方得自己改。

第二件煩人的事是 GitHub 會做套件安全檢查——本意是好事——但它會要你升級 gem，結果就和版型依賴的套件撞在一起，帶來破壞性更新。

最糟的是，一旦你的客製化跟新版套件衝突，乾脆得重來。

而且光為了一個網站，我還得裝 Ruby、Bundler 和一堆 gem，平常根本不會用到。

跟維護整包比起來，這樣也許還算值得，但對我來說還是太重。沒多久我就把網站忘得一乾二淨。

幾年後，我又為了一個小專案（冷氣遙控）對部落格燃起熱情。我換了另一個版型，好不容易發了一篇文章，但整體看起來還是半成品。這次我更沒時間處理其他雜事，因為家裡多了人要照顧。

突然之間，一切都變了。LLM 進展太快，我在工作上已經用它做很多事。有一天我又想起這個網站，因為我決定認真做一件很有興趣的事（那是另一個故事……），想把紀錄放在某個地方。

一開始我還是請 AI 用 Jekyll 架站，但沒過多久我就發現，乾脆請 AI 從頭刻一個網站也行。跟 AI 來回討論了幾輪功能可行性和我想要的視覺效果之後，AI 就開始大改。

大概一兩個小時裡迭了幾次，好了——網站在這裡——自己的風格，純 HTML 和 JavaScript，沒那麼多套件管理，也比較不怕依賴炸掉！最重要的是，之後想加什麼，我都可以自己來，比較不用怕跟外面的東西不相容。

所以……再見了，Jekyll！
